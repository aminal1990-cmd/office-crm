# اتوماسیون اداری و CRM

نسخه‌ی اولیه‌ی یک پنل RTL فارسی برای اتوماسیون اداری و CRM است. پروژه فعلا بدون وابستگی خارجی ساخته شده تا سریع اجرا شود و بعدا بتوان آن را به بک‌اند، دیتابیس و احراز هویت واقعی وصل کرد.

## اجرا

از داخل همین پوشه:

```bash
python3 -m http.server 4173
```

بعد مرورگر را باز کنید:

```text
http://127.0.0.1:4173
```

حساب مدیر اولیه:

```text
admin@demo.local
admin123
```

## امکانات فعلی

- قابلیت نصب روی موبایل به صورت PWA
- فایل manifest و service worker برای اجرای موبایلی و cache پایه
- پایه اعلان مرورگر برای آماده‌سازی نوتیفیکیشن
- نسخه خام بدون مشتری، فرصت یا نامه نمونه
- لاگین و خروج
- مدیریت کاربران، نقش‌های سفارشی و سطح دسترسی قابل انتخاب
- تعریف نقش جدید با نام دلخواه و انتخاب دسترسی‌ها توسط مدیر سیستم
- تنظیمات نام شرکت، شعار، رنگ، فونت، مشخصات ثبتی و اطلاعات سربرگ
- ثبت و ویرایش مشتری
- ثبت لاگ CRM برای هر مشتری: تماس، جلسه، یادداشت و پیگیری
- کانبان فرصت‌های فروش با جابه‌جایی کارت‌ها به صورت کشیدن و رها کردن
- نمودارهای مدیریتی
- داشبورد قابل ویرایش با امکان کم/زیاد کردن بخش‌ها و نوشتن بخش سفارشی
- تعریف تسک از طرف مدیر برای کارکنان و پیگیری وضعیت
- نامه‌نگاری داخلی و اداری با محیط تایپ نامه
- انتخاب گیرنده نامه از بین کاربران فعال سیستم
- پیوست فایل برای نامه‌ها
- پیش‌نمایش و چاپ A4 نامه‌ها
- چت داخلی عمومی و محرمانه بین کارکنان
- انتقال فایل در چت داخلی
- ذخیره‌سازی موقت در `localStorage`

فونت پیش‌فرض روی `Vazirmatn, Vazir, Tahoma` تنظیم شده است. اگر فونت وزیری/وزیرمتن روی سیستم نصب باشد همان استفاده می‌شود؛ در غیر این صورت مرورگر از فونت جایگزین استفاده می‌کند.

## آنلاین کردن رایگان

برای اینکه بدون پرداخت ارزی آنلاین شود، مسیر پیشنهادی فعلی:

1. ساخت یک حساب رایگان در GitHub
2. آپلود پوشه `outputs/office-crm` داخل یک repository
3. اتصال repository به Cloudflare Pages یا Vercel
4. تنظیم مسیر خروجی روی ریشه همین پوشه
5. دریافت آدرس رایگان مثل `*.pages.dev` یا `*.vercel.app`

این مرحله نسخه وب و موبایل PWA را آنلاین می‌کند. برای اینکه اطلاعات بین چند کاربر واقعی مشترک شود، مرحله بعد باید `localStorage` با دیتابیس و API جایگزین شود.

مسیر رایگان پیشنهادی برای مرحله بعد:

- Frontend/PWA: Cloudflare Pages
- دیتابیس و Auth: Supabase Free یا یک PostgreSQL رایگان مشابه
- فایل‌ها و عکس پروفایل: Storage رایگان همان سرویس تا سقف محدودیت رایگان
- Push Notification واقعی: Web Push با Service Worker و HTTPS

نکته مهم: نوتیفیکیشن واقعی بین دستگاه‌ها نیاز به بک‌اند دارد. چیزی که الان اضافه شده، پایه PWA و اجازه اعلان مرورگر است؛ بعد از اتصال دیتابیس و سرور نوتیفیکیشن، پیام‌ها و تسک‌ها واقعاً روی موبایل push می‌شوند.

## اتصال دیتابیس آنلاین رایگان Supabase

این نسخه یک اتصال ساده به Supabase دارد که کل اطلاعات سیستم را در جدول `crm_state` ذخیره می‌کند. تا وقتی تنظیمات فعال نشده، اپ مثل قبل با `localStorage` کار می‌کند.

مراحل:

1. در Supabase یک پروژه رایگان بسازید.
2. از بخش `SQL Editor` فایل `supabase/schema.sql` را اجرا کنید.
3. از بخش `Project Settings > API` این دو مقدار را بردارید:
   - `Project URL`
   - `anon public key`
4. فایل `src/config.js` را ویرایش کنید:

```js
export const SUPABASE_CONFIG = {
  enabled: true,
  url: "PROJECT_URL",
  anonKey: "ANON_PUBLIC_KEY",
  table: "crm_state",
  stateId: "main"
};
```

5. تغییرات را در GitHub آپلود کنید تا Cloudflare Pages دوباره deploy کند.
6. سایت را باز کنید. اگر اتصال درست باشد بالای صفحه عبارت `دیتابیس آنلاین وصل است` نمایش داده می‌شود.

هشدار امنیتی: این اتصال برای شروع و تست آنلاین رایگان ساخته شده است. چون هنوز Auth سروری واقعی ندارد، نباید اطلاعات خیلی حساس داخل آن وارد شود. مرحله حرفه‌ای بعدی، تبدیل همین ساختار به جدول‌های جداگانه با Supabase Auth و سیاست‌های دسترسی دقیق است.

## مسیر توسعه

- جایگزینی `localStorage` با API و دیتابیس
- افزودن گردش کار تایید نامه‌ها و درخواست‌ها با امضای مرحله‌ای
- افزودن حذف نقش و کنترل محدودیت‌های امنیتی پیشرفته برای نقش مدیر
- افزودن پیوست، کامنت و تاریخچه فعالیت برای مشتریان و نامه‌ها
- افزودن اعلان و یادآوری برای پیگیری‌ها
- جایگزینی ذخیره فایل در مرورگر با آپلود واقعی روی سرور
